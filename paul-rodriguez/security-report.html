What Passed

12 Healthy Checks

The baseline security posture is solid. These are the checks that came back clean on a first audit pass.

ChatGPT OAuth (not raw API key)

Authentication uses OpenAI's PKCE OAuth flow through the official Codex CLI. No long-lived API key stored in plain text. Token refreshes automatically under file lock.

Local binding only on gateway

OpenClaw gateway listens on 127.0.0.1:18789 (loopback only). Not reachable from the network or other machines. Verified via Get-NetTCPConnection.

Credentials in user-scoped paths

All secrets live under C:\Users\paul\.openclaw\, readable only by Paul's Windows account. Inherits NTFS ACL protection.

Telegram bot DM allowlist

@PauloradoBot has dmPolicy: "allowlist" with only Paul's Telegram user ID. Any other sender is silently ignored. No public exposure, no pairing-code backdoor.

Approval gates installed

Custom windows-approval-gates skill catches 5 high-risk action categories (outbound messages, deletes, registry, scheduled tasks, downloaded binaries) before they fire. Every gate logs to workspace/logs/approvals.jsonl.

Prompt injection defenses

Agent treats all external content (web pages, incoming messages, pasted files) as potentially adversarial. Summarizes rather than quotes, ignores "ignore previous instructions" patterns, refuses to modify its own config based on external input.

Financial data lockdown

Dollar amounts, account numbers, and trade intent stay in DMs only, never in groups. Built into the approval-gates skill.

Background service is user-level, not SYSTEM

Both scheduled tasks (OpenClaw Gateway, OpenClaw Agent) run as workerbee303\paul, not as a privileged service account. Principle of least privilege.

Local embeddings -- no vector leaks

Your knowledge base is embedded locally by nomic-embed-text. Search queries are embedded locally. The full vector index never leaves the machine. Cosine similarity math runs in PowerShell, in-process.

Enrolled device on HTTPS only

Background agent connects to api.openclawinstall.net exclusively over HTTPS. No fallback to plaintext. Cloudflare-backed cert and HSTS in play.

Fresh OpenClaw version

v2026.4.15 (as of 2026-04-18) -- latest release. Past the 2026.4.14 regression that affected some users.

Backup snapshot before changes

Full .openclaw\ directory was backed up to openclaw-backup-20260417-134410.zip before any modifications. Rollback is always one restore away.

Accepted Trade-offs

3 Known Items, Deferred by Choice

Not every improvement is worth doing today. These three we've noted and accepted as trade-offs.

Telegram bot token stored in plaintext JSON

Currently in ~/.openclaw/openclaw.json at channels.telegram.botToken. Fine for a single-user local setup (the file is user-scoped). The more secure path is OpenClaw's secrets subsystem -- we'll migrate when there's a natural reason to touch the config again.

Windows ExecutionPolicy is per-session Bypass

Scheduled-task-launched PowerShell sessions inherit a stricter execution policy than your interactive session. We work around this by invoking scripts with an explicit -ExecutionPolicy Bypass flag, which is safe but verbose. A one-time Set-ExecutionPolicy -Scope LocalMachine RemoteSigned would be cleaner. Not urgent.

OneDrive-redirected PowerShell profile loads on every command

Your Documents folder is OneDrive-redirected (common on corporate-style Windows setups). Every new shell loads C:\Users\paul\OneDrive\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1, which references oh-my-posh (not installed) and throws a harmless error. Purely cosmetic noise -- doesn't affect security or functionality.

Fixed During This Install

2 Items Hardened Along the Way

Pre-existing workspace was on an outdated OpenClaw

Your box had an unconfigured 2026.4.9 install. Upgraded to 2026.4.15 which resolves known bugs in config validation, Ollama provider handling, and some auth-profile edge cases. Full backup taken first.

Missing Git was silently breaking npm installs

OpenClaw has transitive Git dependencies via npm. Without Git on PATH, any npm install -g openclaw@latest would fail mid-download. Installed Git 2.53.0.windows.3 via winget. Now future updates work transparently.

Recommendations

Things worth doing, eventually

Not urgent, not required, but worth knowing about.

Enable BitLocker if it isn't already on

Your knowledge base, Ollama models, and workspace all sit on disk. BitLocker (or any full-disk encryption) protects everything if the laptop is ever lost or stolen. Windows 11 Pro includes it free.

Use a password manager

Not a P-specific thing, but worth mentioning. Your ChatGPT account is the root of trust for everything P can do with the cloud brain -- it deserves a unique, strong password + 2FA.

Review the approvals log every month

P writes every approval prompt + decision to workspace/logs/approvals.jsonl. Once a month, eyeball what you've been approving. If there's a pattern of unexpected requests, it's a signal something's trying to misuse the agent.

Back up the KB periodically

As you ingest more content, your store.jsonl grows. It's just a text file -- copy it to OneDrive, a USB drive, or a git repo periodically. If the ThinkPad ever dies, you don't lose what you saved.

Tell Ryan if @PauloradoBot ever asks for unexpected approvals

The approval gate is the last line of defense against prompt injection or misbehaving skills. If P ever asks to delete something weird, send an unexpected email, or run a downloaded binary you don't recognize -- that's a signal to pause and ping Ryan.

Bottom line: this install followed the same security practices as commercial deployments. Auth via OAuth (not API keys), local-only binding, user-scoped creds, allowlisted messaging, explicit approval gates, local embeddings, versioned software, and pre-change backups. The three warning items are trade-offs, not holes.
