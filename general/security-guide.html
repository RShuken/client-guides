Security researchers found hundreds of exposed instances in the wild. The most common issues:

  • !
    Exposed Control UI
    Gateway bound to 0.0.0.0 instead of loopback, making the admin interface accessible from the internet.
  • !
    No Gateway Password
    gateway.auth.mode not set, allowing unauthenticated access to the control interface.
  • !
    Reverse Proxy Auth Bypass
    Misconfigured proxies causing external connections to be treated as localhost (auto-trusted).
  • !
    Open DM Policy
    dmPolicy set to "open" allowing anyone to message and control the bot.
  • !
    Plaintext Credentials
    API keys and tokens stored with world-readable permissions (644 instead of 600).

Quick Reference

Security Checklist

Run through this checklist to verify your OpenClaw installation is properly secured.

  • 1
    Update to latest version
    Run npm install -g openclaw then restart. All versions before v2026.1.24-1 have critical RCE vulnerabilities.
  • 2
    Check gateway.bind setting
    Should be "loopback" unless you specifically need remote access. Never expose the Control UI to the internet without auth.
  • 3
    Set gateway auth
    Ensure gateway.auth.mode is "password" or "token", never empty. Use a strong password.
  • 4
    Configure reverse proxy correctly
    If using nginx/Caddy/Traefik, set gateway.trustedProxies to your proxy IPs. Make sure the proxy overwrites X-Forwarded-For headers.
  • 5
    Lock down DM access
    Use dmPolicy: "allowlist" or "pairing". Never use "open" unless you explicitly want public access.
  • 6
    Lock down group access
    Use groupPolicy: "allowlist" with specific group IDs. Avoid "open" in production.
  • 7
    Fix file permissions
    chmod 700 ~/.openclaw && chmod 600 ~/.openclaw/credentials/*
  • 8
    Run the built-in audit
    openclaw security audit --deep (and optionally --fix to auto-apply safe guardrails)

Best Practices

Follow these guidelines to maintain a secure OpenClaw deployment.

üîí

Use Strong Models

Claude Opus 4.5 has the best prompt injection resistance. Avoid weak models for tool-enabled agents.

üì¶

Enable Sandboxing

Run exec tools in Docker containers when possible. Set tools.exec.host to "sandbox" instead of "gateway".

üö¶

Require Exec Approvals

Set ask: "on-miss" or "always" in exec-approvals.json so dangerous commands require human confirmation.

üîë

Scope API Keys

Give each service the minimum permissions needed. Use read-only keys where possible.

üë•

Isolate Sessions

Set session.dmScope: "per-channel-peer" to prevent cross-user context leakage in multi-user setups.

üîÑ

Keep Updated

Check for updates monthly. Run openclaw security audit regularly, especially after config changes.

üìã

Review Skills

Don't install skills from ClawHub blindly. Review the code first. 341 malicious skills were found.

üîê

Protect Your Accounts

Enable 2FA on Telegram/Discord/etc. The allowlist is only as strong as the account behind it.
